How to create a modular distributed web application?

The best way to start is to use an application template (a very simple application that has minimum parts). First click here to see what this simple application looks like. This simple application has 3 parts.

  • Header -- the top part that contains brand, navigation menu and search input.
  • Content container -- middle part where the modules will be loaded into this area.
  • Footer -- the bottom part where will be used to show information like copyright, company name etc.
We use bootstrap 4 to make the layout responsive.

In order to load modules into the content container, we need to know where the modules are. All the modules url are stored in a file called module.json under configurations. When user click a navigation link (menu item in the header), the system will load the module with its url into the content container.Each module can be hosted on any static web server (the server should support cross-origin resource sharing, like GitHub, CodePen, Glitch etc).

There are 2 JavaScript library used for the application.

  • https://vmiis.github.io/api/distribution/vmapi.min.js
  • https://vmiis.github.io/framework/distribution/vmframework.min.js
The vmapi.min.js is used for connecting a database on a cloud server. (If no database function we do not need this). The vmframework.min.js is used for some common functions.

We also put some most common used Html/CSS/JavaScript snippet into files for modules to include. All these files are located in https://vmiis.github.io/components/

Once we understand the fundamental parts of the system, we will dive into how to create a module.
