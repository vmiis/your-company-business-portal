About

What is a module?
    A module is a unit with a specific function. It may contain one HTML file or a group of files (HTML, JavaScript, and CSS) which can be hosted on any static web server under any individual account.
What is a web application?
    A web application is a layout module with a group of other modules linked together.
What is a layout module?
    A layout module is the first module loaded into a web browser. It contains a header (banner), a contents container and a footer.
What is a contents container?
    A contents container is a html div where other modules can be loaded into it.

What is the advantage of a modular distributed web application?
  • Fast and flexible.
  • Modules can be hosted anywhere and maintained by different people with different skills.
  • Modules can be reused in different applications.
  • Building a web application is just creating a layout module and links some functional modules together.
What is remote distributed module?
    A remote distributed module is a module hosted under a person or an organisation cloud account (for example GitHub) and maintained by the account owner.
How to link a remote module?
  • Fully trusted. The module is fully loaded into the system’s main DOM.
  • Trusted but remove all scripts. After removing some tags and all scripts, the module will be loaded into the system’s main DOM.
  • Semi trusted. The module is loaded into the system under a Shadow-DOM. The module’s owner can apply CSS style freely to the modules and the CCS will not affect the other modules in the system
  • Untrusted. The module will be loaded into an iframe sandbox for security and will use HTML5 Messaging for communication with the main DOM.
  • Use as local modules after clone (fork to your account) and check the code security.
