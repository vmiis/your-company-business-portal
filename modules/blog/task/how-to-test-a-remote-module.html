How to test a remote module?

If you have a idea and create a module and want to test, you do not need to modify your application to let your module added in, you can simply to test your module with the following steps.
  • step 1: visit https://www.vmiis.com
  • step 2: click modules
  • step 3: type in remote module's url, then click 'Go'
There are 4 types of module integrations that are described as follow.

DIV
    A remote module will be loaded into the main content container. It will work as a local module, so it can run javascript , be affected by any existing style sheet. Usually the module’s owner is fully trusted and is a member of the development team.
DIV with script filter
    A remote module will be loaded into the main content container, but any script and some unsafe tags will be removed. Usually the module’s owner is trusted but is not a member of the development team. They normally just responsible for some text and basic styles. For example, HR people may want to maintain HR forms download links, marketing people may want to modify some product information, accounting people may want to modify a financial report etc.
Shadow DOM
    Shadow DOM is a new technology which allows a completed HTML module be loaded into the main content container, but it has its own CSS scope. That is the module’s owner can freely apply any CSS rules without affect any other modules in the system. The CSS scope is completed isolated from other parts. This makes the module’s owner be able to style the module in any ways. So far the JavaScript in the shadow module is not isolated, so the module’s owner should be fully trusted.
iFrame
    iFrame is used to fully isolate the module from the system. The module’s owner can do what they want. If the module need to exchange data with the system we will use postMessage method for communication between the system and the module.
